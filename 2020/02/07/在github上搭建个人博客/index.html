在Github上搭建你的个人博客

前言

从前一直是用纸笔的方式记录自己的学习和生活，后来越来越嫌弃自己写字难看，于是记笔记的习惯荒废了很久。最近一位看到一位大佬在博客上写东西，觉得很新鲜，于是找了几篇教程开始了搭建自己博客的过程，在这其中由于各种原因，走了一些弯路也踩了一些坑，所以浪费了不少时间，今天在这里总结一下，算是对浪费了的那些时间有个交代。

特别注明： 本篇基于MacOS操作系统，不过在Win和Linux没什么区别（说什么屁话）

总体思路

Git + hexo + github +Markdown
优点：①几乎免费，学生党友好
②代码要求低，小白友好

由于这三条语句在你更新自己的博客时会经常用到，所以需要讲一下他们的作用
clean 的作用是清除缓存文件(db.json)和已经生成的静态文件(public)。如果不用这条语句，在某些条件下（尤其是更换主题后），你可能会发现自己对站点的更改如何也不生效。
generate可以生成静态文件，比如，你用typora写了一个Markdown格式的文件，那么当你使用generate的时候，hexo会自动地将你的.md文件与你使用的博客主题匹配，并形成博客页面。
deploy 部署文件

特别声明

deploy这一步很可能会报错，这是系统环境造成的，可能的原因貌似是ssh配置有问题，我第一次踩到这个坑时花了很久的时间去解决。但是，在我写这篇博客并重复整个步骤时，却没有发现报错，询问了其他有过自己搭博客经历的同学之后发现，这个报错很可能只会在第一次部署博客时出现。不巧的是我忘却了当时出现了什么报错。
所以，如果你在走到这一步时出现了问题，请联系我，我会及时更新这篇博客，谢谢。
